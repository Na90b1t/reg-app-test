# Документация проекта

## Содержание

- `components/` — краткие описания компонентов интерфейса
- `services/` — описание сервисов приложения

## Состояние функционала (текущая версия)

- ✅ Авторизация и регистрация работают через REST API
- ✅ Бэкенд на Node.js + Express
- ✅ Пароли хешируются с помощью bcrypt
- ✅ JWT токены для авторизации
- ✅ Данные пользователей сохраняются в JSON файле (можно заменить на БД)
- ✅ После регистрации пользователь автоматически авторизуется
- ✅ Текущий пользователь отображается в навигации
- ✅ Реализован выход из системы
- ✅ Автоматическое восстановление сессии при перезагрузке страницы

## Архитектура

### Фронтенд (Angular)
- `AuthService` — сервис для работы с API авторизации
- Использует `HttpClient` для HTTP запросов
- Сохраняет JWT токен в `localStorage`
- Автоматически добавляет токен в заголовки запросов

### Бэкенд (Node.js + Express)
- REST API на `/api/auth/*`
- Хранение данных в `users.json` (для простоты)
- Хеширование паролей перед сохранением
- Генерация JWT токенов при успешной авторизации

## API Endpoints

### POST `/api/auth/register`
Регистрация нового пользователя

### POST `/api/auth/login`
Авторизация пользователя

### GET `/api/auth/me`
Получить информацию о текущем пользователе (требует токен)

Подробнее см. `server/README.md`

## Безопасность

- ⚠️ **Текущая версия:** Секретный ключ JWT хранится в коде (только для разработки!)
- ✅ Пароли хешируются перед сохранением
- ✅ Токены имеют срок действия (24 часа)
- ⚠️ **Для продакшена:** Использовать переменные окружения, HTTPS, реальную БД

## Переход на продакшен

Для реального приложения нужно:
1. Заменить JSON файл на базу данных (MongoDB, PostgreSQL и т.д.)
2. Использовать переменные окружения для секретных ключей
3. Добавить валидацию данных на сервере
4. Настроить HTTPS
5. Добавить rate limiting
6. Добавить логирование

По мере появления функционала добавляйте новые заметки в этот файл или создавайте отдельные документы в каталоге `docs`.
